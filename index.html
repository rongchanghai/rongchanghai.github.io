hexo+github配置

今天用hexo在github上面上面搭建了一个静态博客，这是第一篇文章，操作步骤很简单，但是我就遇到了一个感觉埋的很浅，但是有泥的坑。简单的写一下吧！

步骤一

  • 安装git（必须）、安装node（必须）、注册一个github账号（必须）
  • 默认你上面的都已经安装好了

步骤二

  • 现在本地建一个文件夹 hexo（随便什么都可以）

  • cd 到这个文件夹中，命令执行 npm install -g hexo ,如果你的mac 记得sodu

  • 然后初始化hexo

到现在你的本地静态博客就搭建好了，测试一下
先生成静态页面

本地启动一下

现在本地服务就启动了，在浏览器中可以预览一下 http://localhaost:4000

步骤三

  • 与github建立连接
    在你的github上新建一个respository ，名字一定得是your-user-name.github.io ,你的名字是你的用户名，不是你的昵称，这一点要分清楚

  • 现在需要回到本地来，打开_config.yml文件，文件的最下面deploy 修改为

repository 是从你新建的项目上拷贝过来的，可以是你的ssh地址，也可以是你的https地址。 这里有一个坑就是键和值之间冒号后面必须有一个空格。
记得保存！！！

  • 再需要安装一连接插件

到现在为止连接就已经建立了，你不需要clone，不需要push，只需要下面的命令就可以完成push操作等

  • 部署步骤
    每次部署的步骤，可按以下三步来进行。

一写基本操作
hexo new”postName” #新建文章

hexo new page”pageName” #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

hexo deploy #将.deploy目录部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本
